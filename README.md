Этот репозиторий — моё портфолио учебных Data Science проектов. В нём собраны задачи полного аналитического и ML-пайплайна: от предобработки данных и проверки гипотез до обучения моделей, оценки бизнес-метрик, временных рядов, NLP и компьютерного зрения.

Проекты выполнены в Jupyter Notebook и показывают, как я работаю с данными: формулирую задачу, проверяю качество данных, строю признаки, сравниваю модели, выбираю метрики и объясняю результат с точки зрения бизнеса.

## Ключевой стек

**Языки и среда:** Python, Jupyter Notebook  
**Анализ данных:** pandas, NumPy  
**Визуализация:** matplotlib, seaborn  
**Статистика:** scipy, проверка гипотез, доверительные интервалы  
**Machine Learning:** scikit-learn, Linear/Logistic Regression, Decision Tree, Random Forest, GridSearchCV, Pipeline  
**Gradient Boosting:** CatBoost, LightGBM  
**NLP:** TF-IDF, лемматизация, классификация текстов  
**Time Series:** ресемплирование, лаги, скользящие признаки, TimeSeriesSplit  
**Computer Vision:** TensorFlow/Keras, CNN, ResNet50, ImageDataGenerator  
**Метрики:** Accuracy, F1, ROC-AUC, RMSE, MAE, R², sMAPE

## Что демонстрируют эти проекты

### 1. Умение доводить задачу до бизнес-вывода

В проектах есть не только обучение моделей, но и интерпретация результата: выбор региона с учётом рисков, оценка прибыльности тарифов, анализ поведения клиентов, прогноз спроса и модерация пользовательского контента.

### 2. Полный цикл работы с данными

Во всех проектах используется типовой рабочий процесс Data Science:

1. постановка задачи;
2. загрузка и первичный обзор данных;
3. обработка пропусков, дубликатов и аномалий;
4. исследовательский анализ данных;
5. генерация и отбор признаков;
6. обучение и сравнение моделей;
7. проверка на тестовой выборке;
8. итоговый вывод и рекомендации.

### 3. Работа с разными типами данных

- табличные данные;
- временные ряды;
- тексты;
- изображения;
- бизнес-показатели и финансовые расчёты.

### 4. Осознанный выбор метрик

В проектах используются разные метрики в зависимости от бизнес-задачи:

- **Accuracy** — для базовой классификации тарифов;
- **F1 и ROC-AUC** — для задач с дисбалансом классов;
- **RMSE и MAE** — для регрессии;
- **sMAPE** — для технологического процесса восстановления золота;
- **R²** — для проверки качества линейной регрессии после преобразования данных.
